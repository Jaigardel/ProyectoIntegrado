Rally Fotográfico

Este repositorio forma parte de un proyecto de fin de grado centrado en el desarrollo de una plataforma web para la participación fotográfica. La aplicación permite a los usuarios subir imágenes como parte de un rally fotográfico y ofrece un sistema de votación pública para valorar las contribuciones. Aquí se documentan los recursos técnicos, código fuente y herramientas utilizadas durante su desarrollo.

Etapas del Desarrollo

1. Análisis y diseño

En esta etapa repase en detalle los requisitos mínimos que se propusieron.

Desde un principio pensé en usar PHP como tecnología para el backend pues tenía un proyecto parecido al propuesto.

Exploré distintas opciones y tecnologías y me decante por una web sencilla en HTML, CSS y JS vanilla.

Decidí alamacenar las fotos en SupaBase para mantener la base de datos más ligera.

2. Desarrollo de interfaz

Todo el diseño del frontend está hecho con HTML, CSS y Bootstrap para una experiencia clara y responsive.

La carga de imágenes se hace a través de la URL almacenada en la base de datos.

El diseño es sencillo y funcional, pensado en no tener que hacer muchos clics para llegar al contenido relevante.

3. Programación backend

Lo primero fue crear la base de datos que existía en una máquina virtual (desarrollo).

Después se creó la base de datos en Supabase y se conectó a la aplicación (JS).

Se creó un sistema de autenticación y gestión de usuarios para el registro y login.

Se implementó un sistema de votación, gestión de imágenes y gestión de rallys.

4. Gestión de roles y autenticación

Sistema de login y control de accesos según tipo de usuario (invitado, registrado, admin).

Los usuarios pueden registrarse y acceder a su perfil, donde pueden ver y subir sus imágenes.

Los administradores pueden gestionar los rallys, las imágenes y los usuarios.

Los invitados pueden ver las imágenes y votar, pero no pueden subir imágenes ni crear rallys.

5. Subida y almacenamiento de imágenes

Integración con Supabase para el almacenamiento externo de fotos.

Supabase proporciona código JS para poder gestionar la subida de recursos

Las URL de las imágenes se almacenan en la base de datos a la vez que se suben estas.

6. Pruebas y despliegue

Para poder presentar el proyecto decidí subirlo a un hosting gratuito (Byet).

El despliegue se realizó en el servidor de Byet, donde se subieron todos los archivos y se configuró la base de datos.

Se realizaron pruebas de carga y rendimiento para asegurar que la aplicación funcionara correctamente.
